🎬 Lottie(로티) 애니메이션 활용 가이드

애니메이션을 .json 파일로 뽑아 이벤트 페이지에 삽입하는 방법.
벡터라 깨지지 않고, GIF보다 훨씬 가벼움. (보통 수십 KB).

📁 .json 파일 업로드 경로

서버 경로 \\stagenew\ucl_editor\json\
업로드 후 확인 방법 주소창에 직접 입력해서 {"v":"5.9.0"... 글자가 뜨면 OK
https://aa.aladin.co.kr/ucl_editor/json/파일명.json
⚠️ 반드시 위 경로에 올리세요.

⚙️ 두 가지 사용 방법

방법 1. bodymovin (JS 방식) 알라딘 표준

빈 박스에 JS로 애니메이션을 그려넣는 방식.

<section aria-labelledby="bell-heading"> <h2 id="bell-heading" class="sr-only">새 소식 알림 종</h2> <div class="bell-motion" role="img" aria-label="종이 딸랑딸랑 울리는 애니메이션"></div> </section> <script src="https://cdnjs.cloudflare.com/ajax/libs/bodymovin/5.7.14/lottie.min.js"></script> <script> fetch('/ucl_editor/json/bell.json') .then(function(res) { return res.json(); }) .then(function(data) { document.querySelectorAll('.bell-motion').forEach(function(el) { bodymovin.loadAnimation({ container: el, renderer: 'svg', loop: true, autoplay: true, animationData: data }); }); }); </script>
✅ 인터랙션(클릭·스크롤·호버) 붙이기 편해요. 공유 코드라면 이 방식 추천.

방법 2. lottie-player (태그 방식) 빠르고 간단

<img> 쓰듯이 태그 하나면 끝. JS 한 줄도 안 씀.

<section aria-labelledby="bell-heading"> <h2 id="bell-heading" class="sr-only">새 소식 알림 종</h2> <lottie-player class="bell-motion" src="/ucl_editor/json/bell.json" role="img" aria-label="종이 딸랑딸랑 울리는 애니메이션" autoplay loop> </lottie-player> </section> <script src="https://unpkg.com/@lottiefiles/lottie-player@latest/dist/lottie-player.js"></script>
✅ 단순 자동재생 배너·포인트 아이콘에 추천. 진입장벽이 낮아요.
⚠️ <h2> 문구와 aria-label은 "외형"이 아니라 "전하려는 메시지"로 적으세요. 예) ❌ "종 애니메이션" → ✅ "새 소식 알림 종"

🎨 CSS — 크기 꼭 지정하세요!

크기를 안 주면 0px이라 화면에 안 나타나요. 원본 .json의 가로:세로 비율과 박스 비율을 맞춰야 안 찌그러져요.

.bell-motion {width: 150px; height: 150px; margin: 0 auto; display: block;} @media screen and (min-width: 750px) { .bell-motion { width: 240px; height: 240px; } }

🖱️ bodymovin (JS 방식) 인터랙션 응용 방법

아래 응용은 모두 방법 1 (bodymovin) 전용입니다. <script> 안쪽의 bodymovin.loadAnimation({...}) 블록을 통째로 교체하세요. (추가가 아니에요!)

① 클릭 시 한 번 재생 HTML 그대로

자동 재생을 끄고, 클릭했을 때 처음부터 한 번만 재생.

<section aria-labelledby="bell-heading"> <h2 id="bell-heading" class="sr-only">새 소식 알림 종</h2> <div class="bell-motion" role="button" tabindex="0" aria-label="클릭하면 종이 한 번 울려요"></div> </section> <script src="https://cdnjs.cloudflare.com/ajax/libs/bodymovin/5.7.14/lottie.min.js"></script> <script> fetch('/ucl_editor/json/bell.json') .then(function(res) { return res.json(); }) .then(function(data) { document.querySelectorAll('.bell-motion').forEach(function(bellBox) { var anim = bodymovin.loadAnimation({ container: bellBox, renderer: 'svg', loop: false, // 반복 ❌ autoplay: false, // 자동재생 ❌ animationData: data }); function ringBell() { anim.goToAndPlay(0, true); } bellBox.addEventListener('click', ringBell); bellBox.addEventListener('keydown', function(e) { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); ringBell(); } }); }); }); </script>

② 호버 시 재생/정지 (PC) HTML 그대로

마우스 올리면 재생, 떼면 멈춤. 모바일에선 동작 안 함.

<section aria-labelledby="bell-heading"> <h2 id="bell-heading" class="sr-only">새 소식 알림 종</h2> <div class="bell-motion" role="img" aria-label="마우스를 올리면 종이 울려요"></div> </section> <script src="https://cdnjs.cloudflare.com/ajax/libs/bodymovin/5.7.14/lottie.min.js"></script> <script> fetch('/ucl_editor/json/bell.json') .then(function(res) { return res.json(); }) .then(function(data) { document.querySelectorAll('.bell-motion').forEach(function(bellBox) { var anim = bodymovin.loadAnimation({ container: bellBox, renderer: 'svg', loop: true, // 반복 ⭕ autoplay: false, // 자동재생 ❌ animationData: data }); bellBox.addEventListener('mouseenter', function() { anim.play(); }); bellBox.addEventListener('mouseleave', function() { anim.pause(); }); bellBox.addEventListener('focus', function() { anim.play(); }); bellBox.addEventListener('blur', function() { anim.pause(); }); }); }); </script>

③ 같은 로티 여러 개 HTML도 수정

박스를 여러 개 두고 한 번에 그려넣기. fetch는 한 번만!

<section aria-labelledby="bell-heading"> <h2 id="bell-heading" class="sr-only">새 소식 알림 종 모음</h2> <div class="bell-motion" role="img" aria-label="알림 종 1"></div> <div class="bell-motion" role="img" aria-label="알림 종 2"></div> <div class="bell-motion" role="img" aria-label="알림 종 3"></div> </section> <script src="https://cdnjs.cloudflare.com/ajax/libs/bodymovin/5.7.14/lottie.min.js"></script> <script> fetch('/ucl_editor/json/bell.json') .then(function(res) { return res.json(); }) .then(function(data) { document.querySelectorAll('.bell-motion') .forEach(function(box) { bodymovin.loadAnimation({ container: box, renderer: 'svg', loop: true, autoplay: true, animationData: data }); }); }); </script>

❓ 자주 묻는 질문

Q. 아무것도 안 보여요
1. F12 콘솔 에러 메시지 확인
2. DOCTYPE ... is not valid JSON → .json 경로가 틀린 것. 주소창에 직접 URL 쳐서 글자가 뜨는지 확인
3. 글자는 뜨는데 페이지에 안 보임 → fetch 안 주소 오타 확인 (대소문자, 슬래시)
4. 다 맞는데도 안 보임 → CSS에서 박스 크기 줬는지 확인 (크기 0이면 안 보여요)
Q. lottie-player.js와 lottie.min.js 두 줄 다 써야 하나요?
방법 1만 쓸 거면 lottie.min.js 하나만. 방법 2만 쓸 거면 lottie-player.js 하나만. 안 쓰는 줄은 꼭 빼세요 — 로딩 속도에 영향 줘요.
Q. 로티 .json 파일 어디서 받나요?
After Effects에서 직접 만들거나, LottieFiles.com에서 무료/유료 에셋 다운로드. 외주사에 의뢰할 때는 .json 형식으로 달라고 요청하세요.

💡 SEO & 접근성: 의미 있는 문구 쓰기

aria-label과 sr-only 헤딩에는 "어떤 모션인지"가 아니라 "그 모션이 전하려는 메시지"를 적어주세요. 검색엔진과 스크린리더가 페이지의 의미를 정확히 이해할 수 있어요.

❌ 피해야 할 예시

모션의 "외형"만 설명하는 문구. 페이지의 실제 메시지가 전달되지 않아요.

<h2 class="sr-only">종 애니메이션</h2> <div class="bell-motion" aria-label="종 모션"></div> <h2 class="sr-only">하트 모션</h2> <div class="heart-motion" aria-label="하트 움직임"></div> <h2 class="sr-only">박스 흔들림</h2> <div class="box-motion" aria-label="박스 흔들리는 모션"></div>

✅ 권장 예시

페이지가 전달하려는 "의미"를 적은 문구. SEO·접근성·동료 협업 모두에 유리해요.

<h2 class="sr-only">새 소식 알림 종</h2> <div class="bell-motion" role="img" aria-label="종이 딸랑딸랑 울리는 애니메이션"></div> <h2 class="sr-only">구매 감사 인사</h2> <div class="heart-motion" role="img" aria-label="하트가 두근거리는 애니메이션"></div> <h2 class="sr-only">랜덤박스 응모 안내</h2> <div class="box-motion" role="img" aria-label="랜덤박스가 흔들리며 응모를 유도하는 모션"></div>

📌 핵심 3가지
① <h2 class="sr-only">: 페이지의 실제 메시지를 한 줄로 (예: "새 소식 알림 종")
② role="img": 빈 div를 "이미지"로 인식하게 함 (스크린리더가 그냥 지나치지 않음)
③ aria-label: 어떤 모션인지를 시각장애인 사용자에게 음성으로 전달
