알라딘 이벤트
디자인 시스템 구축

반복되는 이벤트 제작 업무를 줄이고, MD도 직접 사용할 수 있는 이벤트 백오피스를 만드는 것이 목표입니다. 현재는 방향성과 프로토타입 검증 단계이며, 실제 운영 적용은 단계적으로 진행할 예정입니다.

⚠️ 이 문서는 2분기 방향성 및 테스트 현황을 공유하기 위한 자료입니다. 현재 프로토타입 단계이며, 실제 운영 반영 전 내부 검증이 필요합니다.
🎯 Objective & Key Results
OBJECTIVE

알라딘 이벤트 디자인 시스템 구축 & 블록 단위 템플릿 제작으로 디자이너 일상 업무를 줄이고, MD 셀프 운영 기반을 마련한다.

  • 1
    도서팀 MD 선호도 조사 및 템플릿 우선순위 확정 실무에서 가장 많이 쓰이는 이벤트 유형 파악 → 5세트 템플릿 제작 기준 결정
    4월 진행 예정
  • 2
    피그마 디자인 시스템 구축 (Variables + 컴포넌트 라이브러리) 디자인 토큰 정의 완료 → 피그마 Variables 등록 → 블록 컴포넌트화
    토큰 정의 완료 / 피그마 등록 진행 중
  • 3
    블록 단위 이벤트 빌더 프로토타입 완성 및 내부 테스트 조립기 구현 완료 → 디자인팀 내부 테스트 → 개선점 반영
    빌더 프로토타입 완성 / 내부 테스트 예정
  • 4
    AI 배경 이미지 생성 기능 연동 검토 기존 프롬프트 공식을 백오피스에 연결 → Gemini API 연동 프로토타입 검증
    프로토타입 테스트 완료 / 실운영 검토 필요
🔬 현재까지 테스트 완료한 기능
블록 조립 빌더 테스트 완료
히어로 4종 / 도서 슬라이더 / 사은품 2종 블록을 드래그 없이 버튼으로 조립. ASCX 파일 추출까지 가능.
AI 배경 이미지 생성 테스트 완료
색상톤 / 분야 / 무드 / 스타일 선택 → Gemini API로 PC+모바일 이미지 자동 생성. 송지연 디자이너 프롬프트 공식 기반.
디자인 토큰 정의 완료
색상 / 타이포 / 간격 / 반경 등 전체 토큰 정의 완료. 피그마 Variables 형식에 맞는 JSON 준비.
네이밍 컨벤션 완료
피그마 ↔ 코드 간 1:1 네이밍 규칙 확정. color/hero-surface ↔ --ds-color-hero-surface
반응형 미리보기 테스트 완료
빌더 내 PC / 모바일 전환 미리보기. 수정 즉시 반영.
피그마 Variables 등록 진행 예정
토큰 JSON → 피그마 Variables 실제 등록. 이후 컴포넌트 라이브러리 구축으로 이어짐.
📅 2분기 일정
4월 — MD 설문조사 어떤 템플릿이 필요한지 현장 파악.
우선순위 및 주제별 템플릿 방향 결정.
4~5월 — 피그마 + 템플릿 제작 Variables 등록 → 컴포넌트화 →
각자 담당 주제 템플릿 디자인.
6월 — 템플릿 코드화 목표 템플릿 디자인 → 빌더 블록 코드화까지 완료 목표.
단, 6월은 창립기념일 이벤트 대응으로 일상 업무 집중도가 높아
이 프로젝트에 온전히 집중하기 어려울 수 있어 일정 변동 가능성 있음.
3분기 — 실운영 검증 실제 이벤트에 적용하면서 문제점 파악.
피드백 반영 후 시스템 고도화.

🎨 AI 배경 이미지 생성 — 테스트 현황

송지연 디자이너의 프롬프트 공식을 기반으로 빌더에 Gemini API를 연동했습니다. 현재 프로토타입 테스트 완료 단계이며, 실운영 도입 여부는 추가 검토가 필요합니다.
